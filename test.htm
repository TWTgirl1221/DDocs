Overview

Keep your transactions simple and PCI compliant with Forte.js. Adding Forte.js to your HTTPS-hosted site reduces your PCI exposure by posting sensitive customer information directly to Forte's servers instead of your own. Forte.js also simplifies your payment processing by ensuring you don't have to worry about the complex security implications that arise when credit card or echeck data hits your server. Instead, after receiving a customer POST, Forte returns a short GUID, single-use token to you, which you can use to create a transaction or a permanent payment token.

Forte.js Workflow

Integrating Forte.js

You can use Forte.js with your own payment form or use Forte's customizable, third-generation payment form, Forte Checkout. The simple integration only requires a few lines of code.

Step 1: Generate an API Login ID

Forte.js requires a merchant-specific API Login ID to accompany the POST call to the Forte servers. To generate this ID, complete the following steps:

  1. Log into your Virtual Terminal Account.
  2. Navigate to the Gateway Settings tab and click the Gateway Key icon (). The following screen displays:





  3. Click next to the API Login ID field. The application generates a new value.

  4. Record this newly generated value in a secure location.

  5. Click to save this value to your merchant account.

Step 2: Include Forte.js in the Payment Form

Add the following Forte.js scripts to the <head> tag of your payment form.

For the test environment:

<script type="text/javascript" src="https://sandbox.forte.net/api/js/v1"></script>

For the production environment:

<script type="text/javascript" src="https://api.forte.net/js/v1"></script>

Step 4: Send the Request

Send the payment information via a POST request using the following required parameters.

Parameter Description Type
card_number The customer's credit card number string
expire_year The expiration year of the customer's credit card [max length = 4] string
expire_month The expiration month of the customer's credit card [max length = 2] string
cvv The card verification value of the customer's credit card [max length = 5] string

Step 5: Receive the One-Time Token Response

The AJAX callback that Forte sends in response to the request echoes back all the data sent in the request (save for the card_number parameter). The table below displays the available response parameters:

Parameter Description Type
onetime_token The one-time use token that Forte returns (e.g., ott_452343423450)

NOTE: One-time tokens expire after 10 minutes.

 string
last_4 The last four digits of the customer's credit card number string
expire_year The expiration year of the customer's credit card [max length = 4] string
expire_month The expiration month of the customer's credit card [max length = 2] string
cvv The card verification value of the customer's credit card [max length = 5] string
event

Indicates whether or not Forte successfully created the one-time use token. Use one of the following values for this parameter:

  1. success
  2. failure
string

The code sample below displays a successful event response.

{"event":"success","response_code":"A01","response_description":"APPROVED","onetime_token":"OTT_1OTOETSERSSRSLHV64S8MG","card_type":"VISA","last_4":"1111","cvv":"123","expire_year":"2015","expire_month":"01"}
